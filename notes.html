Difference between CHAR and VARCHAR

CHAR

  • Stores data in fixed length.
  • If the allocated space is not fully used, remaining space is filled with blank spaces.
  • Also called a static data type.
  • Defined size of CHAR is fixed (e.g., CHAR(2000)).
  • Can store up to the defined number of characters.
  • Execution is faster.

VARCHAR

  • Stores data in variable length.
  • Only actual data is stored; unused space is not consumed.
  • Also called a dynamic data type.
  • Default size varies (e.g., VARCHAR2 in Oracle up to ~4000 bytes depending on settings).
  • Uses memory efficiently.
  • Execution is slower compared to CHAR.

FROM Clause

  • FROM clause starts execution.
  • In the FROM clause we pass the table name as an argument.
  • The job of the FROM clause is to go to the database, search the table, and put the table under execution.
  • If the table does not exist in the database, it throws the error: table or view does not exist.

SELECT Clause

  • SELECT clause starts execution after the FROM clause.
  • In SELECT clause we can pass three types of arguments:
    • * (all columns)
    • Column names
    • Expressions
  • SELECT clause goes to the table under execution and selects the mentioned columns.
  • SELECT clause is responsible only for reporting the result table (output table).

Note: Semicolon (;) is used to terminate the query.

Function

function are a block of code or set of instruction which used to perform specific tasks
there are three main components of a function
  1. Function_name
  2. Number of arguments (no of inputs)
  3. Return type
In function we have two types :
  1. User Defined function
  2. In built function
    1. Single Row function
    2. Multi Row function

Single row function

  • SRF() Execute Row by Row
  • SRF takes one input process and executes it to generate an output and takes next input and so on
  • if we pass 'n' number of input to the single row function it returns 'n' number of output

Multi Row function

  • MRF execcutes Group by group
  • IT takes all the inputs at one shot and then executes and provides A single Output
  • If we pass 'n' number of input to MRF() it return '1' output
Notes
  1. MRF() accepts only one arguments (ex: max(sal))
  2. we cannot use any column along with MRF() in select clause
  3. Count is only one MRF() which can accept '*' as an arguments
    • count() is used to find the number of rows in the given table or given column
  4. MRF() ignores null values
  5. we cannot write MRF() as a condition in where clause

Difference Between Single Row Function (SRF) and Multi Row Function (MRF)

Also Called Scalar Function / Server Function Aggregate Function
